Hashing and Consistent Hashing (Cache) — Notes
1. Hashing
Definition

Hashing is a technique used to map data (keys) to servers using a hash function.

Formula:

server = hash(key) % N

Where:

key = data identifier (userId, sessionId, etc.)

N = number of servers

Example

Servers:

Cache1
Cache2
Cache3

Keys distribution:

Key	Hash	Server
user1	21	Cache1
user2	35	Cache2
user3	50	Cache3
Problem with Normal Hashing

When a server is added or removed, the value of N changes.

Example:

Before:

hash(key) % 3

After adding server:

hash(key) % 4

Result:

Most keys move to different servers

Massive cache miss

Database overload

2. Consistent Hashing
Definition

Consistent hashing distributes servers and keys on a hash ring to minimize data movement when servers are added or removed.

Instead of:

hash(key) % N

We place servers and keys on a circular hash ring.

Hash Ring Concept

Example ring:

0° → 360°

Servers placed on ring:

Cache1 → 50
Cache2 → 150
Cache3 → 250

Keys are also hashed and placed on ring.

Rule:

➡ A key is assigned to the first server clockwise on the ring.

Example
KeyA → 120

Clockwise server:

Cache2

So:

KeyA → Cache2
Adding a New Server

Add:

Cache4 → 200

Now only keys between 150 → 200 move.

Benefits:

Minimal data movement

Better scalability

Stable caching system

3. Virtual Nodes
Problem in Consistent Hashing

Servers may receive uneven traffic.

Example:

Cache1 --- Cache2 -------------- Cache3

Cache3 may handle most keys.

Solution: Virtual Nodes

Each physical server appears multiple times on the ring.

Example:

Cache1-A
Cache1-B
Cache1-C

Cache2-A
Cache2-B
Cache2-C

Cache3-A
Cache3-B
Cache3-C

Ring becomes:

C1A → C2A → C3A → C1B → C2B → C3B → C1C → C2C → C3C

Now keys distribute evenly across servers.

Benefits of Virtual Nodes

Better load balancing

Even distribution of keys

Improved fault tolerance

Easier scaling

4. Real Systems Using Consistent Hashing
System	Usage
Redis Cluster	Data partitioning
Memcached	Distributed caching
Cassandra	Data distribution
DynamoDB	Partitioning

Problem:

When servers change, most keys move.

Keys assigned to next clockwise server

One-Line Interview Answer

Consistent hashing distributes keys and servers on a hash ring so that when servers are added or removed, only a small subset of keys are reassigned. Virtual nodes further improve load balancing by representing each physical server multiple times on the ring.


Q&A


✅ 1. What is Caching?

Caching is a technique of storing frequently accessed data in a fast storage layer (usually memory) so that future requests can be served faster without hitting the database.

✅ 2. Why do we use Cache?

We use cache to improve performance by reducing latency, decreasing database load, and increasing system scalability.

✅ 3. What is Cache Hit vs Cache Miss?

Cache Hit

When requested data is found in cache and returned directly.

Cache Miss

When data is not found in cache, so the system fetches it from the database and stores it in cache for future use.

✅ 4. What are Eviction Policies?

Eviction policies define how cache removes data when memory is full.

Common types:

LRU (Least Recently Used) → removes least recently accessed data

LFU (Least Frequently Used) → removes least frequently accessed data

FIFO (First In First Out) → removes oldest data

✅ 5. What is Consistent Hashing?

Consistent hashing is a technique that distributes data across servers using a hash ring, ensuring that when servers are added or removed, only a small portion of data is redistributed.

✅ 6. What are Virtual Nodes?

Virtual nodes are multiple logical representations of a physical server on the hash ring, used to evenly distribute data and improve load balancing.

✅ 7. What is Cache Invalidation?

Cache invalidation is the process of updating or removing stale data from cache when the underlying data in the database changes.

✅ 8. What is Cache Stampede?

Cache stampede occurs when multiple requests simultaneously try to fetch data from the database after a cache miss or expiration, causing heavy load on the database.


🔥 1. Cache Expiry (TTL - Time To Live)
What is it?

Cache expiry means data in cache is automatically removed after a certain time.

Example:

User data cached for 5 minutes

After 5 minutes → data is deleted or refreshed.

Why needed?

Prevent stale data

Keep cache updated

Example
Cache:
user:101 → "Jeevan" (TTL = 300 sec)

After 300 seconds → expired

Interview Answer

Cache expiry (TTL) ensures that cached data is automatically invalidated after a defined time to avoid stale data.

🔥 2. Consistent Cache
What is it?

A consistent cache ensures that the data in cache and database are synchronized (same data).

Types of Consistency
Strong Consistency

Cache always matches DB

Hard to maintain

Eventual Consistency (most used)

Cache may be slightly outdated

Eventually becomes correct

Example
DB: "Jeevan Kumar"
Cache: "Jeevan"

After some time → cache updates

Interview Answer

Consistent cache ensures data in cache reflects the database, typically achieved using eventual consistency in distributed systems.

🔥 3. Write Policy (Very Important)

Defines how data is written to cache and database.

3 Types
1️⃣ Write Through
App → Cache → Database

Both updated at same time
✔ Consistent
❌ Slightly slower

2️⃣ Write Back (Write Behind)
App → Cache → DB (later)

✔ Very fast
❌ Risk of data loss if cache fails

3️⃣ Write Around
App → Database (skip cache)

✔ Avoid unnecessary cache
❌ First read is slow

Interview Answer

Write policies define how data is written between cache and database, such as write-through, write-back, and write-around.

🔥 4. Least Recently Used (LRU)
What is it?

LRU removes the data that has not been used for the longest time.

Example

Cache size = 3

Access order:
A → B → C

Access A:
B → C → A

Add D → remove B
Why useful?

Assumes recent data is more important

Widely used in Redis

Interview Answer

LRU evicts the least recently accessed data when cache is full.

🔥 5. Least Frequently Used (LFU)
What is it?

LFU removes the data that is accessed the least number of times.

Example
Key	Count
A	10
B	2
C	5

👉 Remove B

Why useful?

Keeps frequently used data

Better for stable access patterns

Interview Answer

LFU evicts data with the lowest access frequency when cache is full.

Concept	Key Idea

🔥 Final Interview Summary (Use This)

Caching involves managing data freshness using TTL, ensuring consistency between cache and database, defining write policies like write-through and write-back, and handling memory limits using eviction strategies like LRU and LFU.
